IDM

Inter Domain Messaging

“IDM is a virtual-network protocol that supports peer-to-peer communication between devices connected to any type of network. IDM integrates addressing for node, process and resource in a single identifier. The transport and native network protocols can be avoided in many situations”

Over many protocols

Despite attempts to use IP everywhere, many vendors and technologies are still using other domain-dependent protocols. IDM can use these underlying protocols to reach the destination node. Over ZigBee, nRF24, RS485, RS232, BlueTooth... or any other protocol that can carry a message between two remote or neighbor peers.

Drop complexity

Using the correct endpoints, you don't need to care about conections, sockets, resources, handlers... Just focus on writing your application, and let IDM handle the communications part! Also, using ZeroC Ice's intuitive IDL (called Slice), it's easy to define the contract between clients and servers: 

module IoT {
    interface WaterPump {
        void on();
        void off();
    };
};

Talk to everything

Thanks to its flexible addressing scheme, you can add as many objects, devices or nodes as you need. IDM defines a variable-lenght address, from 16 to 128 bits, you choose what suits your needs.

Ready for constrained devices

IDM requires a very low footprint. Both services and clients may be implemented in highly resource-constrained devices, as low as Class 0 hardware (like Arduino UNO, 32 KiB code, 2 KiB RAM, see RFC 7228).

Made to connect your devices

IDM is built upon ZeroC Ice. With implementations for C++, C#, Java, JS, Objective-C, PHP, Python and Ruby, its easy to fit in almost every platform. And what about constrained devices? You can use our IceC (Ice for Constrained devices), available for Arduino, ESP8266 and other platforms.
